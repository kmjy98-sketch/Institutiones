## B. 룰 (실증·강제)

### B-1. 근거 (기존 R1·R4·R6 + #34 적용례)
1. **기록 밖 사실 창작 금지**: 모든 사실 인용은 문제 [사실관계] 항 번호로 (`(제6항)`). 상담 요지 등 당사자 일방 진술과 "다툼 없는 사실"을 구분한다.
2. **실재 판례만**: 인용 판례 전건을 국가법령정보센터 API로 검증한다(korean-law-mcp, 미연결 시 직접 호출 폴백). 검증 실패 판례는 인용하지 않는다.
3. **판례 문구 원문 정합**: 직접 인용("…")은 API 반환 원문의 연속 부분문자열만. 패러프레이즈 시에도 원문 핵심 표지어("저당권의 목적되는", "행하여진 것으로 본다", "패소한 때에는…악의의 점유자로 본다" 등)를 유지하고 충분조건/필요조건을 뒤집지 않는다.
4. **순환 인용 금지**: 중간 산출물(md)을 후속 산출물의 근거로 재인용하지 않는다. 항상 추출본(원 기록)에서 재확인.

### B-2. 정확성 (이번 작업에서 실제 잡힌 오류 유형)
5. **합계 금액은 구성 명시**: 이종 항목 합산액을 단일 명목으로 부르지 않는다 (원리금 4억 6,800만 + 비용 100만 = "원리금 등 합계 4억 6,900만").
6. **용어-사실 정합**: 등기·지목 등 공부상 상태와 모순되는 관용어 금지 (지목 '답'인 토지에 "나대지" 금지 → "건물이 존재하지 않는 토지").
7. **절차 행위는 대상 명시**: "즉시항고하지 않았다" → "그 임의경매개시결정에 대하여 즉시항고하지 않았다". 등기 말소도 원인 그대로("강제경매로 인한 매각을 원인으로").
8. **주술 호응·대비 구문**: 무주어 술어 금지, "위 사안은 ~인 반면, 이 사건은 ~" 대비 정렬, 대시(—) 삽입구 대신 "즉, ~ 중 어느 때에도".

### B-3. 논증 구조
9. **다중 안전망 구조**: 핵심 쟁점은 주위적 구성이 무너져도 결론이 유지되도록 층화한다 (예: 담보가등기 → 가사 순위보전이라도 선행 가압류 기준 → 가사 압류 기준이라도 건물 부존재).
10. **상대 최강 논거 먼저**: 반박부는 상대가 원용할 가장 강한 재판례부터. 구별(distinguishing)은 ①사실관계 차이 ②후행 대법원 판례 존재 순으로.
11. **불리 판례 정면 처리**: 회피하지 않고 사안 차이로 적용 배제 논증 (96다31895 → 합의해제 사안 vs 원시 차용).

## C. 파이프라인 (실증 순서)

1. **일괄 추출 (토큰 0)**: 기록 PDF 전건 → pdfplumber → `sync/대회/_추출/{사건ID}/*.md`. docx 지시문은 python-docx, hwp는 olefile `PrvText`(미리보기 ~3,000자). 기존 추출본 있으면 재추출 금지.
2. **전수/우선 정독 분기**: 합계 약 100면 이하면 전수 정독.
3. **사실관계 정리**: 타임라인 + 쟁점 매핑. 문제에 "다툼 없다" 명시 시 불일치표 생략.
4. **조문·판례 API 검증**:
   - 조문: `lawService.do?target=law&MST={법령}&JO={조번호6자리}` (예: 366조 = `036600`)
   - 판례: 사건번호 직접 검색이 1순위. 키워드는 0건이 흔함 → 단어 수 축소 재시도.
   - **검색 결과 상단의 유관 사건명을 반드시 훑는다** — 목표 판례 검색 중 더 최신·더 직접적인 판례가 발견될 수 있음 (2010다52140 검색 → 2024다236327 발견 사례).
   - 연결 오류는 재시도(backoff 2~6초)로 흡수.
5. **서면 작성**: 권장 목차(사안의 핵심 → 쟁점별 주장 → 상대 주장 반박 → 결론), 판례는 본문 괄호 인용, 분량 설계는 "11pt·160%·A4 ≈ 면당 1,300~1,500자" 기준으로 초안 길이를 제한 분량의 95% 이내로.
6. **docx 생성·검증 루프**:
   - docx-js 스크립트로 생성. **npm 패키지는 C: 로컬에 설치** (H:/구글드라이브 경로는 npm install 깨짐).
   - Word COM(PowerShell)으로 PDF 변환 + `ComputeStatistics(2)` 면수 확인 (LibreOffice 불필요).
   - 면수 초과 시 압축 우선순위: ①문단 간격 ②상하 여백(20mm까지 — 한글 기본) ③자구 압축(의미 무손실) ④줄간격 155%까지(기준 "160% 내외" 한도) ⑤서명·날짜 블록 통합. 1행 초과 = 약 50자 컷.
   - 최종 PDF에서 인용 전수 검증: **공백 정규화 후 substring 매칭** (PDF 줄바꿈 때문에 원문 그대로는 오탐).

## D. 형식 기본값 (지시문 우선, 없으면 이 값)

- 글꼴 바탕 11pt / 줄간격 160%(초과 시 155%까지) / A4, 상하 20mm·좌우 25.4mm
- 제목 "참 고 서 면" 16pt 굵게 가운데 / 본문 양쪽 맞춤
- 서명·작성 명의·날짜는 **문제 지시문 표기와 글자 단위로 일치** (예: "법무법인 민재 담당변호사 박평등")
- 산출물: `sync/대회/{사건ID}/01_사실관계정리.md, 02_쟁점조문판례.md, 03_답안_{유형}.md, {제출파일명}.docx/.pdf`
