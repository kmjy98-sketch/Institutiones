# 기록형 풀이 룰 (R1~R10)

**R1. 기록 한정 (Record Grounding)**
사실관계는 제공된 사건기록에 적힌 것만 사용한다. 새로운 사실을 가정하거나 기록을 임의로 보강하지 않는다. 기록형 채점은 "기록에 있는 사실의 누락 없는 활용"이 핵심이며, 기록 밖 사실 창작은 즉시 감점 사유다.

**R2. 미확인 보류 (Not-in-Record)**
기록에서 확인할 수 없는 단정은 출력하지 않는다. "기록에서 확인할 수 없습니다" 또는 `[자료 부족—보류]`. 보류가 잘못된 단정보다 항상 낫다.

**R3. 인용 의무 (Citation Mandatory)**
- 사실 단정: `[《문서명》 p.X | "정확한 발췌"]` (기록 쪽수·증거번호 병기)
- 법리 단정: 조문 원문 또는 판례 사건번호+판시 발췌
- 위치 식별자 없으면 `(확인 불가—위치 식별자 부족)`

**R4. 실재 판례만 (No Fabricated Authority)**
존재가 확인되지 않은 판례·조문·논문을 인용하지 않는다. 기억에 의존한 사건번호는 `[검증필요]` 표기 후 확인 경로(종합법률정보 사건번호 검색, korean-law-mcp)를 안내한다. 조문번호·사건번호는 Claude Code 환경에서는 korean-law-mcp로 자동 검증한다(CLAUDE.md #13).

**R5. 원문 우선 (Original-Text Priority)**
조문·판례는 원문 우선. 판례의 태도는 원문에 확인된 표현만 사용하고 사건번호를 병기한다(CLAUDE.md #34). 원문이 없으면 `(원문 미포함)`.

**R6. 순환 인용 금지 (No Self-Citation)**
직전 응답·이전 단계에서 모델 자신이 만든 요약·표·합성 문장을 다음 단계의 출처로 재인용하지 않는다. 출처는 항상 기록 원문 또는 공식 원문(법령정보센터·대법원·헌재·KCI)이다. 1단계 정리표를 3단계에서 인용할 때도 인용 표기는 정리표가 아니라 **원 기록 문서**를 가리켜야 한다.

**R7. 한자→한글 (Korean Output)**
甲→갑, 乙→을, 條→조. 외국어 법률용어는 한국어 대역 우선, 필요 시 괄호 보조. §·①②③ 기호는 유지.

**R8. IRAC 변형 골격 (기록형 특화)**
쟁점 → 법리(조문·판례) → 포섭(기록 발췌 인용) → 소결. 포섭이 기록형의 핵심: 포섭 문장마다 기록 인용을 붙인다. 민사는 요건사실 순서(권리근거→항변→재항변, 2열 표는 CLAUDE.md #32), 형사 변호인 서면은 공소사실 요지→쟁점별 의견→정상관계→결론.

**R9. 상대방 주장 예상·반박 (Adversarial Completeness)**
상대방이 주장할 청구원인·항변·재항변을 최소 1개 예상하고 반박을 서면 내부에 배치한다. 불리한 쟁점은 회피하지 않고 최소한의 방어 논리를 제시한다. 자기 입장에 불리한 판례의 구별(distinguishing) 논리도 평가 대상.

**R10. 추론 비노출·서면 문체**
내부 추론은 출력하지 않는다. 최종 서면은 법원 제출 문체("~입니다")로, "조사해보니"·"~인 것 같다" 등 메모·추측 문체를 금지한다.

## 인용 표기 표준 (요약)

- 판례: `대법원 YYYY. M. DD. 선고 NNNN다XXXXX 판결` / 약식 `대판 YYYY.M.DD, NNNN다XXXXX`
- 헌재: `헌법재판소 YYYY. M. DD. 선고 YYYY헌마XXX 결정`
- 조문: `「형법」 제257조 제1항` (정식) / `형법 §257①` (노트형)
- 기록: `[《피의자신문조서(이정한)》 p.7 | "발췌"]` — 문서명에 작성자·차수 병기
