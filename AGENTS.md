# AGENTS.md — 기록형/대회 풀이 운영 기준 (lean)

> 이 repo는 기록형(사건기록) 풀이·참고서면 작성 전용이다. 학습 워크스페이스의 카드·위키·OCR·파일분류 룰은 적용하지 않는다.
> 세션 시작 시: 본 문서 → `rules/기록형풀이_룰.md` → `prompts/` 순으로 읽는다.

## 1. 근거 규칙 (최우선)

1. **기록 한정**: 사실관계는 제공된 사건기록에 적힌 것만 사용. 새 사실 가정·기록 임의 보강 금지.
2. **미확인 보류**: 기록/소스에서 확인 불가한 단정은 출력하지 않는다. "기록에서 확인할 수 없습니다" 또는 `[자료 부족—보류]`. 추측 표현("~인 것 같다") 금지.
3. **인용 의무**:
   - 사실: `[《문서명》 p.X | "정확한 발췌"]` (쪽수·증거번호 병기)
   - 법리: 조문 원문 또는 판례 사건번호 + 판시 발췌
   - 위치 식별자 없으면 `(확인 불가—위치 식별자 부족)`
4. **실재 판례·조문만**: 미확인 사건번호·조문·논문 인용 금지. 기억 의존 사건번호는 `[검증필요]` 표기 후 검증(korean-law-mcp 또는 국가법령정보센터·종합법률정보).
5. **원문 우선**: 조문·판례는 원문 우선. 판례 태도는 원문 확인 표현만 + 사건번호 병기. 원문 없으면 `(원문 미포함)`.
6. **순환 인용 금지**: 모델 자신이 만든 요약·표를 다음 단계 출처로 재인용 금지. 출처는 항상 원 기록 또는 공식 원문.
7. **사건번호 연도 2자리**: `85다카1131`(O) / `1985다카1131`(X). 선고일 미확인 시 추측 금지.

## 2. 출력 형식

- **IRAC 변형(기록형 특화)**: 쟁점 → 법리(조문·판례) → 포섭(기록 발췌 인용) → 소결. 포섭 문장마다 기록 인용을 붙인다.
- **민사**: 요건사실 순서(권리근거→항변→재항변). 청구-항변 구조는 2열 표(`| 원고(갑) | 피고(을) |`), 빈 셀 `—`.
- **형사 변호인 서면**: 공소사실 요지 → 쟁점별 의견 → 정상관계 → 결론.
- **상대방 주장 예상·반박**: 상대가 펼칠 청구원인·항변·재항변을 최소 1개 예상하고 반박을 서면 내부에 배치. 불리한 판례는 구별(distinguishing) 논리 제시.

## 3. 언어·문체

- 한국어. 한자→한글(갑을병정무, 條→조). 외국어 법률용어는 한국어 대역 우선(필요 시 괄호 보조). `§`·`①②③` 기호는 유지.
- 내부 추론 비노출. 최종 서면은 법원 제출 문체("~입니다"). 메모·추측 문체("조사해보니"·"~인 듯") 금지.

## 4. 파일·보안

- 비밀·자격증명(`.env`·키·`mcp.json`)은 커밋하지 않는다(`.gitignore` 적용).
- 예시 사건기록은 마스킹된 가상 자료다. 외부 공개·공유 금지(repo는 Private 유지).
- 파일 삭제 금지. 정리 필요 시 `_trash/`로 이동.
